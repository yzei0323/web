웹서버란? 웹을 제공하는 프로그램, 컴퓨터를 말해요.. 서빙이란 말 많이 들어 봤죠 .. 제공하는것 !, 웹페이지를 제공하는 컴퓨터입니다.

여러분이 지금 보시는 웹페이지가 바로 제가 제공하는 웹페이지입니다.

예시로) 네이버 서버는 많은 만화페이지와 연예뉴스 페이지를 가지고 있다가 요청하면 요청한 클라이언트에게 (요청한 웹브라우저 ) 만화페이지를 제공합니다.

인터넷이 연결된 상태에서 웹브라우저를 통해서 네이버에 페이지를 요청 (주소 : url )하고 네이버 서버가 주는 데이터를 받아 웹브라우저가 랜더링 한 결과를 보고 있습니다.!!

    http://www.mysite.com:80/edu/index.html

    위 url 요청의 의미?

      www.mysite.com 도메인에 대한 IP주소 찾기 (DNS 서버) www.mysite.com 이용해 컴퓨터 찾기 TCP/IP
    • 80포트로 실행중인 서버 프로그램을 찾는다
    • edu/index.html 사용하여 클라이언트가 요청한 문서를 찾아 실행한 결과 또는 그 문서 자체를 응답한다
    • 정적문서는 그 문서 자체를 응답한다
    • 동적문서는 실행한 결과를 응답한다

    HTTP 란?

    웹브라우저와 웹서버가 통신할 때 서로 약속된 요청, 응답메시지를 주고 받으면서 통신이 이루어진다
    Http는 신뢰성있는 통신과 처리 효율이 높은 프로토콜로서 인터넷 환경에서 가장 적합한 통신구조이다

    1. 무연결 :요청이 있을 때마다 독집적으로 연결하여 통신하는 방식
    2. 무상태: 이전 연결에서 했던 작업을 다음요청에서 사용할 수 없다. , 즉 상태 정보가 유지되지 않는 특성을 말한다 : 서버의 부담이 적다 (웹사이트의 많은 사용자가 옴)
    3. 요청/응답구조 요청해야 응답한다 , 그냥 응답하지는 않는다.
    4. 웹브라우저 와 웹서버
    5. 요청과 응답
    6. HttpServletRequest HttpServeltResponse
